В данной работе реализована схема из двух контейнеров (Web-приложение на Python и база данных Postgres), работающих в изолированных сетях.

### Ключевые особенности:
* **Изоляция:** Использованы две разные bridge-сети (`web_net` и `db_net`).
* **Безопасность:** Контейнеры не имеют прямой связи друг с другом на сетевом уровне.
* **M3 Support:** Конфигурация адаптирована для работы на Apple Silicon (использованы slim-образы).

### Как запустить:
1. Перейти в папку лабы: `cd lab1`
2. Запустить контейнеры: `docker-compose -f docker-compose.good.yml up -d`

### Проверка изоляции:
При попытке пинга из контейнера `web` в сторону `db`:
`docker exec -it docker_lab1-web-1 ping db`
Система выдает ошибку `Name or service not known`, что подтверждает успешную работу изоляции DNS и сетей.
